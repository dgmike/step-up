StepUp

StepUp is a command-line tool that helps other projects to manage your versioning increment.

Check the RubyGems Page View on GitHub

Automating semantic version increment

Introduction

Basic usage:

gem install step-up
Successfully installed step-up-0.9.5
1 gem installed
cd your_repo_root/repo_name
stepup
v0.2.0+3

Consider that your project has a tag named "v0.2.0". The "+3" part of the output above means the project has three commits since last version tag.

According to SemVer.org, we create this utility to manage the version increment based on the rules of semantic versioning (with some customized rules). Considering the default settings of StepUp, we have the following version mask:

v0.5.3.2 => vMAJOR.MINOR.PATCH.TINY

Each level of a semantic version corresponds to the scope of magnitude of changes that your project suffered. The increment of each level is based on the following:

  • MAJOR version when you make incompatible API changes (Example: architecture changes, inauguration version, and so on...),
  • MINOR version when you add functionality in a backwards-compatible manner (new features, changes in external dependencies, or something like),
  • PATCH version when you make backwards-compatible bug fixes or small adjustments, and
  • TINY version when you make minor changes (like documentation changes, for example).

Version increment

If you want to make available a new release of your project, then this utility will assist you in the step of increasing its version. The versioning increment is based on the last version tag found in the repository, and works as follows:

v0.5.3.2 => v0.5.3.3 (TINY increment)
v0.5.3.2 => v0.5.4 (PATCH increment)
v0.5.3.2 => v0.6.0 (MINOR increment)
v0.5.3.2 => v1.0.0 (MAJOR increment)

As you can see, the TINY level is omitted when its value is zero. The mask allows you to configure the less relevant levels this way.

The command-line example of a version increment:

cd your_repo_root/repo_name
stepup
v0.5.3.2+3
stepup notes
---
Bugfixes:

  - Fix some encodding problems
stepup version create --no-editor

The command of the example above forces the auto-increment of the version level according to the notes found in the GIT repository. To understand how it work, see the Creating enriched release notes effortlessly page.

This utility wraps some git-commands to automate the usage of the SCM objects (like tags, branches and notes). The git-notes are archived into the version tag, and you can make rich changelogs with them, and present them with the format you will.

stepup changelog -n1 --format=wiki
== v0.5.4 (Aug/02 2014 13:36 -0300 by kawamanza) ==

Bugfixes:

* Fix some encodding problems
stepup changelog -n1 --format=rpmspec
* Sat Aug 02 2014 by kawamanza - v0.5.4

  Bugfixes:

    - Fix some encodding problems

StepUp has a set of commands to assist the organization and control in the project development.
